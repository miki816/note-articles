# 毎日のAI記事自動生成スキル

## 概要
毎日自動実行。最新AI情報をリサーチし、note用の記事を生成してファイルに保存する。
筆者が手動でネタを指定した日はそちらを優先する。

---

## Step 0: 手動ネタの確認

note-articlesフォルダに `today_topic.txt` がある場合：
- その中身をテーマとして使う
- URLが含まれている場合はそのページを取得して内容を把握する
- Step 1〜2をスキップしてStep 3に進む
- 記事生成後、today_topic.txtを自動削除する

`today_topic.txt` の例：
```
Claude Code Agent Teamsの使い方を解説したい。有料記事で。
```
```
https://x.com/xxx/status/12345 このポストの内容を深掘りして記事にして。
```
```
https://youtube.com/watch?v=xxx この動画のテーマで記事を書いて。無料記事で。
```

today_topic.txt がなければ、Step 1から自動リサーチを開始する。

## Step 1: 自動リサーチ（情報源ごとに検索）

- 「ITmedia AI 最新」で検索

### 英語AIニュースアグリゲーター

---

## Step 2: テーマ選定

収集した情報から1つのメインテーマを選ぶ。選定基準（優先度順）：

1. **日本のユーザーに直接影響がある**（日本語対応、価格変更、日本での利用開始等）
2. **個人事業主・副業者がすぐ実務に使える**
3. **まだ日本語で詳しく解説されていない**（英語圏のニュースを先取り）
4. **SNSでバズっている / 話題性がある**
5. **読者が「お金を出してでも知りたい」と思える具体性がある**

選べるニュースがない日は、以下のストックテーマから選ぶ：
- Claude Code活用Tips（CLAUDE.mdの書き方、スキル設計等）
- AI画像生成の最新ワークフロー
- AIを使った副業の具体例と収益モデル
- 複数AIツールの比較レビュー
- AI時代のSEO戦略

---

## Step 3: 記事カテゴリ・価格判定

| カテゴリ | 価格 | 判定基準 |
|---|---|---|
| ニュース速報・解説 | 無料 | 新ツール・アップデート・リリースの解説。流入を増やす目的 |
| 使い方・テンプレート | 500円 | 具体的な手順・プロンプト・設定値を含む |
| 副業・マネタイズ | 500円 | AIで稼ぐ方法の具体例 |
| 比較レビュー | 無料 or 500円 | 概要なら無料、詳細な比較表や使い分け指南は有料 |
| 失敗談・学び | 500円 | 体験ベースの希少情報 |

基本方針：週7記事のうち、無料3〜4本 / 有料3〜4本のバランスを目指す。

today_topic.txt に「有料記事で」「無料記事で」の指定がある場合はそれに従う。

---

## Step 4: 記事生成

### 必須ルール（CLAUDE.mdから）

**文体：**
- 一人称「自分」または省略
- 一文は短く。40文字以内を目安
- 読点は1文に1〜2個まで

**絶対禁止：**
- 「いかがでしたでしょうか」
- 「〜ではないでしょうか」
- 「〜と言っても過言ではありません」
- 「さて」「ところで」で段落を始める
- 「AIの力を借りて」「AIをフル活用して」の多用
- 「誰でも簡単に」「最短で」「爆速で」の煽り
- 「まとめ」「おわりに」セクション名
- 見出しに絵文字
- 運用中サービスの具体名・URL
- 前職の具体的な会社名

**文字数ガイドライン（note公式データに基づく）：**

無料記事：1,500〜2,500文字
- 読者に「このアカウントは質が高い」と思わせるための入口
- 短すぎると薄く見え、長すぎると毎日投稿で読者が疲れる

有料記事（500円）：合計3,000〜4,000文字
- 無料エリア：1,200〜1,600文字（「何を解決できるか」の価値提示）
- 有料エリア：1,800〜2,500文字（具体的な手順・プロンプト・設定値の核心部分）
- 根拠：note公式データで有料エリアの文字数と売上の相関はほぼゼロ。実用ノウハウ系は「無料部分を厚く書いて価値を伝え、有料部分はコンパクトに核心を突く」のが勝ちパターン
- 有料部分が1,000文字未満だと「500円でこれだけ？」と不満が出る。1,800文字以上を確保すること

**構成：**

無料記事（1,500〜2,500文字）：
```
タイトル（数字 or 疑問形を含む）
導入（3〜5行）
本文
締め（明日やること / 次に試すこと）
ハッシュタグ
```

有料記事（500円、合計3,000〜4,000文字）：
```
タイトル（数字 or 疑問形を含む）

[無料エリア：1,200〜1,600文字]
導入 → 概要（無料部分だけでも「へぇ」となる情報） → 有料で書く内容リスト

--- ここから有料 ---

[有料エリア：1,800〜2,500文字]
具体的な手順・プロンプト・設定値 → 結果 → 応用ポイント → 次にやること

ハッシュタグ
```

## Step 5: ファイル保存

ファイル名: `YYYY-MM-DD_article.md`
保存先: note-articlesフォルダ直下

ファイル先頭にメタ情報を記載：
```
---
date: YYYY-MM-DD
title: [記事タイトル]
category: [ニュース解説 / 使い方・テンプレート / 副業・マネタイズ / 比較レビュー / 失敗談・学び]
price: [無料 / 500円]
status: 自動生成（要レビュー）
sources:
  - [参考URL1]
  - [参考URL2]
topic_source: [自動リサーチ / 手動指定]
---
```

---

## Step 6: 完了サマリー出力

```
📝 本日の記事生成完了

テーマ: [テーマ]
タイトル: [タイトル]
カテゴリ: [カテゴリ]
価格: [無料 / 500円]
文字数: [約X文字]
ネタ元: [自動リサーチ / 手動指定]
ファイル: ~/note-articles/YYYY-MM-DD_article.md

⚠️ noteにコピペする前に確認してください：
- 導入と締めが自分の言葉になっているか
- AI臭い表現が残っていないか
- 有料記事の場合、無料エリアだけでも価値があるか
```
